某漫画请求头sc参数实现和response解密

2022-07-15 | 编程
文章目录
  1. 1. 请求头sn参数
  2. 2. 响应体解密

最近要工作中需要抓取某漫画的所有章节信息，只能抓取app的信息，但是该app的请求头有个sc参数是随着请求每次变化的，而且返回来的每个response都是加密的，需要解密才能获取。

请求头sn参数

通过重放观察，sc参数每次都是变化的。而且跟localtime、qimei有关

jadx反编译apk搜索sc关键字。

正好搜索到的就是往header中添加sc参数的，追进去看下。

有locatime跟qimei参数，使用objection验证一下

响应体解密

刚开始一直没有找到头绪，后来在一篇文章中看到了相关的请求分析。

找到解密操作的相关实现

刚开始遇到DES3有点懵圈，在python中没法还远，使用了rpc进行解密。

返回来的是bytes类型，需要bytes(data).decode()转换成str。

使用rpc调用apk有点不稳定，经常会崩掉

一个在线DES加解密的网站

后来又重新研究了一下python的DES，可以使用python直接解密。

hook相关方法拿到DES的key值，因为没有偏移量，所以也就不用iv值了。

网上某大佬的python实现

可以参照这个实现对响应体的解密操作，需要注意的是Crypto库不能使用，需要安装pycryptodome

pip install pycryptodome
